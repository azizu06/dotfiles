Vim Motions & Neovim Cheatsheet

Terminal-based power editing for cmux + Neovim + LazyVim

🌱 Beginner: Core Motions (Day 1–3)

Learn these first. Everything else builds on them.

Modes

i → Insert mode (edit before cursor)
a → Insert mode (edit after cursor)
Esc or jj or jk → Back to normal mode
v → Visual mode (select text)
Shift-V → Visual line mode

Navigation (hjkl are your arrows)

h ← j ↓ k ↑ l → Left, down, up, right
w → Jump to next word start
b → Jump to previous word start
e → Jump to word end
0 → Beginning of line
$ → End of line
gg → Top of file
Shift-G → Bottom of file

Edit (delete, yank, paste)

d + motion → Delete (e.g., dw = delete word)
dd → Delete entire line
y + motion → Yank/copy (e.g., yw = copy word)
yy → Yank entire line
p → Paste after cursor
Shift-P → Paste before cursor
u → Undo
Ctrl-r → Redo

Example: Your First Edit

Let's say you want to change a word:

  1. Move to the word: w or b
  2. Delete it: dw
  3. Type the new word in insert mode: i
  4. Exit: Esc

Or shorter: cw (change word) — combine delete + insert in one motion.

⚡ Intermediate: Efficiency Hacks (Week 1)

Find & Replace Within Line

f + char → Find char forward (e.g., fa jumps to first 'a')
Shift-F + char → Find char backward
; → Repeat last find forward
, → Repeat last find backward
Example: f( jumps to next (, then ; for the next one

Text Objects (The Secret Sauce)

ci + char → Change inside (e.g., ci( = change inside parentheses)
di + char → Delete inside
ca + char → Change around (including the delimiters)
da + char → Delete around
Huge example: ciw (change in word), cip (change in paragraph), ci" (change in quotes)

Line Jumps

Shift-H → Jump to top of visible screen
Shift-M → Jump to middle of visible screen
Shift-L → Jump to bottom of visible screen
zz → Center current line on screen

Counts (Repeat Motions)

3w → Jump 3 words forward
5j → Move down 5 lines
4fa → Find the 4th 'a'
With relativenumber on, you see line distances, making counts natural: "that line is 6 down, so 6dj"

Search (Global Motions)

/ + text → Search forward
? + text → Search backward
n → Next match
Shift-N → Previous match

🔥 Advanced: Power Chains (Month 1+)

Combine motions, operators, and text objects into one-key editing cascades.

Operators (Composable Prefixes)

c → Change (delete + insert)
d → Delete
y → Yank (copy)
> → Indent
< → Dedent
= → Auto-format (reindent)

Compound Examples

6dj → Delete next 6 lines
c3w → Change next 3 words
y2e → Copy to the end of next 2 words
dip → Delete paragraph
>% → Indent matching bracket pair
== → Auto-format current line

Special Motions

% → Jump to matching bracket/paren/brace
. → Repeat last action (incredible for repetitive edits)
Ctrl-v → Visual block mode (select columns)

The Dot Operator (Repeat)

Do an edit once, then . repeats it at your next cursor position.

Example: Change one variable name with cw, move to the next with w, then hit . to repeat the change. Chain: cw → type → Esc → w → . → w → .

📝 Neovim + LazyVim Specifics

Leader Key (Space)

Hit space and wait — which-key pops up showing all available commands.

Space space → Fuzzy find files
Space / → Grep (search text in all files)
Space e → Toggle file explorer
Space o → Open Obsidian vault (search notes, create links)
Space s v → Vertical split
Space s h → Horizontal split
Space b d → Close current file (delete buffer)
Space q l → Open quickfix list (errors/warnings)
Ctrl-s → Save (auto-formats on save)

Window Navigation

Ctrl-h ← Ctrl-j ↓ Ctrl-k ↑ Ctrl-l → Move between splits
Shift-H / Shift-L → Prev/next buffer (open file)

LSP (Language-Aware Features)

gd → Go to definition
gr → Find all references
K → Hover (show docs/type info)
Space cr → Rename (variable/function across file)
[d / ]d → Previous/next diagnostic (error)

Autocomplete

Start typing, blink.cmp shows suggestions automatically
Ctrl-n / Ctrl-p → Next/prev suggestion
Ctrl-y → Accept suggestion
Ctrl-e → Dismiss menu
Completions are per-language (JS knows imports, Python knows dependencies, etc.). All from your installed LSP servers.

Obsidian Integration

Space o → Open Obsidian picker
Type to fuzzy-search your vault
Ctrl-v (in picker) → Create new note
In-file: wiki-links like [[Note Name]] auto-complete with Ctrl-n
You can edit your Obsidian notes inline in Neovim without switching apps.

🎮 Practice & Learning Resources

Free, gamified ways to practice Vim motions until they're muscle memory.

Interactive Games

🎮 VimAdventures — Browser-based game. Learn hjkl with level progression. 30 min to basics. (Free tier covers essentials) ⛳ Vim Golf — Coding puzzles solved with minimal keystrokes. Race against others' solutions. Highly addictive. ⚡ Speedcoder — Race-style typing with Vim commands. Real-time multiplayer. Gamifies speed.

Learning Paths

📖 Vim Tutor — Built-in. Run vimtutor in terminal. 30 min, official Vim tutorial. Best starting point.
📚 Vim Wiki — Community docs. Dry but comprehensive. Good for reference after basics. 🎥 Learn Vim Progressively — Structured written guide. Top-rated for self-paced learning.

Recommended Practice Schedule

  1. This week (Day 1–3): vimtutor (30 min) + VimAdventures 1 hr. Goal: hjkl + i/a/Esc + dw/yy/p reflexive.
  2. Week 1–2: Edit real code in Neovim. Use cheatsheet open beside. Focus on w/b/e, f/, ci(, daw chains.
  3. Week 2–4: 10 min daily Vim Golf. Practice text objects + counts. Motions become automatic.
  4. Month 1+: No cheatsheet. Vim motions are flow. Move to advanced chains (6dj, cip, >%). Explore plugins (which-key keeps learning fun).

Key insight: 20 hours of deliberate practice (games + real editing) → 10x faster than mouse-based editors. The learning curve is front-loaded; the payoff lasts forever.

Pro Tips for Learning

🎯 Use relative line numbers. They make counting distances natural. See :set rnu (already on in LazyVim).
🎯 Disable arrow keys while learning (forces hjkl). After 1 week, re-enable if you want — by then hjkl is reflexive.
🎯 Practice the dot (.) operator daily. Edit once, repeat with . = muscle memory chaining.
🎯 Keep which-key menu visible. Explore commands by hitting Space. Curiosity-driven discovery is faster than reading docs.

When You Get Stuck

Q: I know the motion but forgot the key combination.

A: Hit Space and wait for which-key. Scroll visually to find what you're looking for.

Q: I want to do X, what's the Vim way?

A: In real editing, use Vim Golf to see how experts solve it. Or use Space / to search the docs.

Q: Am I doing this inefficiently?

A: After month 1, check your most-used motions. If you're reaching for the mouse often, that's your next target to practice.
